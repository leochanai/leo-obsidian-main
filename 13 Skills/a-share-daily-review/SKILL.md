# A 股每日复盘

从财经网站获取当日市场数据，生成结构化的 Markdown 复盘报告。

## 自动化抓取器（长期可复用）

本 Skill 已内置一个 **API-first** 的抓取脚本，用于稳定生成复盘报告骨架（指数/成交额/涨跌停/炸板/板块/连板等）。

- 脚本路径：`scripts/a_share_daily_review.py`
- 输出路径：默认写入 `15 Reports/YYYY-MM-DD A股复盘.md`
- 原则：**拿不到的数据写“数据暂缺”，不编造**

### 用法

在 Obsidian vault 根目录执行：

```bash
# 生成今天的复盘（默认输出到 15 Reports/）
python3 "scripts/a_share_daily_review.py"

# 生成指定日期的复盘
python3 "scripts/a_share_daily_review.py" --date 2026-01-22

# 自定义输出路径（不覆盖已有文件，除非加 --overwrite）
python3 "scripts/a_share_daily_review.py" --date 2026-01-22 --out "15 Reports/2026-01-22 A股复盘 (auto).md" --overwrite

# 如果遇到上游接口限流/连接被重置导致“看起来卡住”，用 fast 模式先生成骨架
python3 "scripts/a_share_daily_review.py" --date 2026-01-22 --fast

# 控制概念板块分页抓取规模（默认 6 页）
python3 "scripts/a_share_daily_review.py" --max-concept-pages 3

# 用 JSON 文件填充“重要消息”（policy/funding/market）
python3 "scripts/a_share_daily_review.py" --news-json "references/news-example.json"

# 默认会自动抓取“重要消息”（标题+链接，best-effort）；如需禁用：
python3 "scripts/a_share_daily_review.py" --no-auto-news
```

## 步骤 1：获取市场数据

### 数据获取策略

按优先级尝试：

| 优先级 | 方法 | 说明 |
|--------|------|------|
| 1 | **自动抓取脚本** | 直接跑 `scripts/a_share_daily_review.py`，生成可用的“数据骨架” |
| 2 | **Web Search** | 用于补齐脚本覆盖不到的字段（政策/事件/资金面解读） |
| 3 | **财经门户** | 作为兜底或交叉验证（新浪、东方财富、证券时报等） |

### 推荐搜索查询

```
# 使用 websearch_web_search_exa 工具
query: "A股 今日行情 复盘 涨跌停 {YYYY-MM-DD}"
numResults: 5
```

### 需要收集的数据

| 类别 | 具体内容 |
|------|----------|
| 大盘指数 | 上证指数、深证成指、创业板指、科创50 收盘价及涨跌幅 |
| 涨跌统计 | 涨停/跌停数量、上涨/下跌家数、炸板率 |
| 成交数据 | 两市总成交额、北向资金净流入 |
| 热点板块 | 涨幅前5板块、跌幅前5板块及领涨/领跌个股 |
| 龙头个股 | 连板股、人气股、市场辨识度高的股票 |
| 重要新闻 | 政策面、资金面、消息面的重要信息 |

### 备用数据源

如 Web Search 结果不足，可尝试直接抓取：

| 来源 | URL | 主要内容 |
|------|-----|----------|
| 东方财富 | https://www.eastmoney.com/ | 大盘指数、热点板块 |

## 步骤 2：分析市场行情

基于收集的数据进行分析：

### 大盘分析
- 三大指数表现（涨跌幅、成交量变化）
- 技术面分析（支撑/压力位、均线趋势）
- 量能分析（缩量/放量、量价配合）

### 板块分析
- 主线逻辑：当日领涨板块的驱动因素
- 资金偏好：主力资金流入方向
- 板块轮动：与前一日对比的变化

### 市场情绪
- 赚钱效应：涨停家数、连板高度
- 亏钱效应：跌停家数、炸板率
- 市场人气：成交额、换手率

### 后市展望
- 短期关注点
- 潜在风险
- 操作建议（标注"仅供参考，不构成投资建议"）

## 数据源与覆盖范围（脚本）

脚本当前主要使用 **东方财富公开接口**（`push2*`/`push2ex*`）来获取：

- 指数收盘价/涨跌幅/成交额（支持按日期取历史 kline）
- 成交额（以沪深两市及北证50成交额聚合近似“沪深京三市成交额”）
- 涨停/跌停/炸板数量与封板率（按日期取）
- 行业板块涨跌幅排名（当日快照）与板块领涨个股（当日快照）
- 涨停池中按连板数排序的“连板龙头”列表（按日期取）

仍可能出现 `数据暂缺` 的字段：

- 北向资金净流入（目前未找到稳定的免费口径，优先保证不编造）
- “主力资金净流入/净流出板块”等更细分资金口径（可后续迭代补充）
- 重大新闻（建议用 Web Search/人工摘录，避免脚本自动拼接造成噪音）

## Troubleshooting

- 运行时报错 `Refusing to overwrite existing file`：加 `--overwrite` 或换一个 `--out` 文件名。
- 某些字段为 `数据暂缺`：说明当前数据源无法稳定获取该字段；优先用 Web Search 补齐，再考虑扩展脚本。

### 保存位置

```
15 Reports/YYYY-MM-DD A股复盘.md
```

如 `15 Reports/` 目录不存在，先创建该目录。

### 数据缺失处理

如某项数据无法获取，在报告中标注"数据暂缺"，不要编造数据。

## 注意事项

- **最佳执行时间**：交易日 15:30 后（收盘数据已更新）
- **日期确认**：默认使用当前日期，如需历史复盘需用户明确指定
- **数据完整性**：如数据不完整，在报告开头标注缺失部分
