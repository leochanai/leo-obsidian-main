# 内容转 HTML 可视化网页

将文章内容转换为专业的中文可视化HTML网页，强调设计美感与内容完整性的平衡。

## 工作流

复制此清单并跟踪进度：

```
任务进度：
- [ ] 步骤 1：确定美学方向
- [ ] 步骤 2：选择字体和配色
- [ ] 步骤 3：设计内容结构
- [ ] 步骤 4：处理格式和细节
- [ ] 步骤 5：生成完整 HTML
```

---

## 步骤 1：确定美学方向（编码前必须完成）

分析内容后，明确选择一个大胆的美学方向：

| 决策项 | 说明 |
|--------|------|
| **Purpose** | 页面解决什么问题？目标读者是谁？ |
| **Tone** | 选择一个鲜明风格 |
| **Differentiation** | 用一句话描述设计的独特之处 |

**风格选项**：
- 极简主义 / 极繁主义 / 复古未来 / 有机自然 / 奢华精致 / 俏皮玩具感
- 编辑杂志风 / 粗野主义 / 装饰艺术 / 柔和粉彩 / 工业实用主义

---

## 步骤 2：设计系统

### 字体

**禁用**：Inter, Roboto, Open Sans, Lato, Arial, 系统默认字体

**推荐**：

| 风格 | 字体 |
|------|------|
| 代码 | JetBrains Mono, Fira Code, Space Grotesk |
| 编辑 | Playfair Display, Crimson Pro, Newsreader |
| 技术 | IBM Plex 系列, Source Sans 3 |
| 独特 | Bricolage Grotesque, Syne, Outfit |

**配对原则**：高对比度 = 有趣（展示字体 + 等宽字体，衬线 + 几何无衬线）

**使用极端**：字重 100/200 vs 800/900，字号跳跃 3x+

### 色彩

**禁用**：紫色渐变 + 白色背景的陈词滥调

**要求**：
- 使用 CSS 变量管理配色系统
- 主导色 + 锐利强调色，胜过平均分配的怯懦配色

**灵感**：IDE 主题（Dracula, Nord, Catppuccin）、杂志设计

### 动画

**核心**：一个精心编排的页面加载动画（带 animation-delay 的错落出现）比零散微交互更令人愉悦

- 优先使用纯 CSS 动画
- 重点：滚动触发效果、出人意料的悬停状态
- 避免：到处都是但没有章法的动画

### 背景与视觉深度

**禁用**：纯色背景作为默认选择

**创造氛围**：渐变网格、噪点纹理、几何图案、叠加透明层、戏剧性阴影、装饰性边框、颗粒感覆盖

### 空间构图

- 意想不到的布局、不对称设计
- 元素重叠、对角线流动
- 打破网格的元素
- 大方的负空间或有控制的密度
- 标题与上方内容间距 > 标题与下方内容间距

### 图标

**禁用**：Emoji 表情符号（✅ ❌ 📌 💡）

**推荐图标库**：
- Phosphor Icons: https://phosphoricons.com
- Lucide Icons: https://lucide.dev

通过 CDN 引入 SVG 图标或内联 SVG，全站统一使用同一图标库。

## 步骤 3：内容结构化

根据内容类型智能识别并匹配展示形式：

| 内容类型 | 展示形式 |
|----------|----------|
| 对比/比较 | 并列卡片或对比表格 |
| 流程/步骤 | 带编号的流程图或时间线 |
| 列举要点 | 特色图标列表 |
| 重要引用/模板 | 突出显示的引用卡片 |
| 案例/示例 | 分步骤的展开式布局 |

---

## 步骤 4：格式处理

- 反引号（`）内容 → 代码样式
- `#` 开头标签 → 彩色标签组件
- 保持列表层级关系
- 代码块 → 专门的代码展示组件
- 清理所有原始格式标记

---

## 步骤 5：输出要求

生成完整的独立 HTML 文件：

1. 包含所有内嵌的 CSS 和 JavaScript
2. 响应式设计，适配各种屏幕
3. 顶部创建吸引人的标题区域
4. 保持内容完整性和逻辑性
5. **每次生成必须不同**：变换明暗主题、不同字体、不同美学
6. 确保禁用 JavaScript 后，所有内容仍可见可读

---

## 核心原则

### 内容完整性

> **设计服务于内容，不为美观丢失信息。**

| 行为 | 说明 |
|------|------|
| **保留** | 核心观点、关键数据、重要示例、完整列表项 |
| **禁止** | 遗漏关键信息、用"等"替代具体列举、过度概括 |
| **允许** | 精简冗余表述、优化语句结构、调整排版 |

### 技术稳健性

- 静态渲染优先
- 渐进增强
- 避免 JS 依赖的内容切换
