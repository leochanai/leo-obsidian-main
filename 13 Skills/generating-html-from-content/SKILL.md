# 将文章或内容转换为专业的中文可视化HTML网页

使用 Gemini 3.0 Pro 将以下文章内容转换为一个专业的中文可视化HTML网页。
## 第零步：内容完整性原则

> **核心原则：设计服务于内容，不要为了美观而丢失重要信息。**

- **保留**：核心观点、关键数据、重要示例、完整的列表项
- **禁止**：遗漏关键信息、用"等"替代具体列举、过度概括导致信息丢失
- **允许**：精简冗余表述、优化语句结构、调整排版提升可读性
## 第一步：确定美学方向（必须在编码前完成）

分析内容后，明确选择一个大胆的美学方向并坚定执行：
- **Purpose**: 这个页面要解决什么问题？目标读者是谁？
- **Tone**: 选择一个鲜明的风格（从以下选择或自创）：
- 极简主义 / 极繁主义 / 复古未来 / 有机自然 / 奢华精致 / 俏皮玩具感
- 编辑杂志风 / 粗野主义 / 装饰艺术 / 柔和粉彩 / 工业实用主义
- **Differentiation**: 这个设计有什么令人难以忘怀的独特之处？用一句话描述。

## 第二步：设计原则

### 字体（Typography）
- **禁止使用**: Inter、Roboto、Open Sans、Lato、Arial、系统默认字体
- **推荐选择**:
	- 代码风格: JetBrains Mono、Fira Code、Space Grotesk
	- 编辑风格: Playfair Display、Crimson Pro、Newsreader
	- 技术风格: IBM Plex 系列、Source Sans 3
	- 独特风格: Bricolage Grotesque、Syne、Outfit
- **配对原则**: 高对比度 = 有趣。展示字体 + 等宽字体，衬线 + 几何无衬线
- **使用极端**: 字重 100/200 vs 800/900，字号跳跃 3x+，而非保守的 1.5x

### 色彩与主题（Color & Theme）

- **禁止**: 紫色渐变 + 白色背景的陈词滥调
- **要求**: 使用 CSS 变量管理配色系统
- **策略**: 主导色 + 锐利强调色，胜过平均分配的怯懦配色
- **灵感来源**: IDE 主题（Dracula、Nord、Catppuccin）、文化美学、杂志设计
### 动画与交互（Motion）

- **核心原则**: 一个精心编排的页面加载动画（带 animation-delay 的错落出现）比零散的微交互更令人愉悦
- **优先使用**: 纯 CSS 动画方案
- **重点时刻**: 滚动触发效果、出人意料的悬停状态
- **避免**: 到处都是但没有章法的动画

### 背景与视觉深度（Backgrounds & Visual Details）

- **禁止**: 纯色背景作为默认选择
- **创造氛围**: 渐变网格、噪点纹理、几何图案、叠加透明层、戏剧性阴影、装饰性边框、颗粒感覆盖
- **匹配整体美学**: 背景处理必须与选定的风格方向一致
### 空间构图（Spatial Composition）

- 意想不到的布局
- 不对称设计
- 元素重叠
- 对角线流动
- 打破网格的元素
- 大方的负空间 或 有控制的密度
- 标题与上方内容之间的间距 > 标题与下方内容之间的间距（视觉上"标题属于下方内容"）

### 图标（Icons）

- **禁止使用**：Emoji 表情符号（如 ✅ ❌ 📌 💡）
- **推荐图标库**：
	- Phosphor Icons: https://phosphoricons.com
	- Lucide Icons: https://lucide.dev
- **使用方式**：通过 CDN 引入 SVG 图标或内联 SVG
- **风格一致性**：全站统一使用同一图标库，保持线宽、风格一致

## 第三步：内容结构化（智能识别）

- 对比/比较内容 → 并列卡片或对比表格
- 流程/步骤 → 带编号的流程图或时间线
- 列举要点 → 特色图标列表
- 重要引用/模板 → 突出显示的引用卡片
- 案例/示例 → 分步骤的展开式布局

## 第四步：格式处理

- 正确处理 Markdown 语法：
* 反引号（`）内的内容处理为代码样式
* 识别 # 开头的标签并转换为彩色标签组件
* 保持列表的层级关系
- 清理所有原始格式标记，只保留内容
- 代码块用专门的代码展示组件呈现

## 第五步：技术稳健性原则

- 静态渲染优先
- 渐进增强
- 避免 JS 依赖的内容切换
## 输出要求

1. 生成完整的独立 HTML 文件，包含所有内嵌的 CSS 和 JavaScript
2. 响应式设计，适配各种屏幕
3. 顶部创建吸引人的标题区域
4. 保持内容的完整性和逻辑性
5. **每次生成必须不同**: 变换明暗主题、不同字体、不同美学，避免收敛到相同的选择
6. 确保禁用 JavaScript 后，所有内容仍可见可读
