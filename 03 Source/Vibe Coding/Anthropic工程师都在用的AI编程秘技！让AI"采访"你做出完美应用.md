# Anthropic 工程师都在用的 AI 编程秘技！让 AI “采访”你做出完美应用

![](https://www.youtube.com/watch?v=iyXUf33JG_g)

**00:00** 大家好，谢谢。今天我将介绍一个非常好用的 AI 编程工具技巧，你可以用在 Claude Code 里面，也可以用在 Codex、Cursor 或者 Augment 里，都行。

**00:11** 这个技巧是 Anthropic 的工程师分享的。他说他喜欢用 Claude Code 构建应用的方式是通过规范（Specification），从一个最小的规范或 Prompt 开始，然后使用 `Ask User Question` 工具要求 Claude 采访你，最终生成一个完整且详尽的规范文档。

**00:31** 简单来说，就是让 AI 阅读这个规范文档，并使用工具对用户进行详细的访谈，内容可以涵盖任何方面。这里的 `Ask User Question` 工具，其实就是 Claude Code 之前推出的让用户进行选择的多选面板工具。

**00:51** 我们先来看一下通过这段提示词，让它做一个 macOS 应用。虽然说这个工具目前只有 Claude Code 原生支持，但你将这个指令放在 Augment、Cursor 或者 Codex 里面，它们通常也能理解你的意思。

**01:10** AI 知道你想要做什么，它会开始对你进行访谈。它想了解一些技术细节：目标平台是 iOS、macOS 还是多平台？设备适配如何？屏幕方向、烟火效果的核心视觉是什么？在我回答之后，它又问了我第二部分：技术实现。

**01:31** 涉及粒子系统与物理模拟，接着是“新年快乐 2026”的文字展示。然后是交互设计，接着它开始第三部分：视觉细节与体验流程、背景设计、时间线与流程设计等等。还有烟花的发射逻辑，虽然这个应用可能看起来简单，但涉及的细节还是非常多的。

**01:56** 再之后，我们看第四部分：交互细节和技术考量，包括退出界面、控制项目以及代码偏好。接着来到第五部分：最终确认。控制栏有哪些功能可以选择，还有一些边缘情况。它总共给出了 39 个问题，问得特别详细。最终，它生成了一个完整的规范文档。

**02:23** 之后，我新开了一个窗口，让 o1-pro 根据文档生成应用，并提示它可以使用 Xcode Build MCP。这个 MCP 特别好用，在让 AI 帮我构建这个 macOS 应用的过程中，我全程没有打开 Xcode 程序。

**02:41** 过程中它主要用到了这样的一个 MCP。它同样还问了我几个问题，关于音效、上海天际线是否有一些图片资源，以及整个项目的规模。接着它列出了 9 个计划。我们可以看到，它调用了 Xcode Build MCP 先将项目创建成功。

**03:04** 然后开始创建核心文件。中间如果出现错误，它会自动重新创建。之后，它就开始尝试构建了。构建过程中，它用到了多个工具，会自动发现错误并自行修复，不需要我打开 Xcode 点击运行，再发现问题反馈给它。

**03:25** 到了这里，它就完成了前两个阶段。当时已经能看到烟花的效果了，当然效果还比较简单。接着是第三阶段——物理引擎。之后，它说这个应用完成了，并提示点击任何位置即可发射烟花，鼠标移动到右边缘会显示控制面板。

**03:45** 之后我发现烟花的类型比较少，背景里也看不清剪影。我就让它进行修复：我希望给上海的天际线加一些灯光效果，而且要加大烟花的亮度和火花。它又修改了一下。

**04:12** 之后我提示它：“新年快乐 2026”文字和樱花（烟花）效果要大一些，颜色也要多彩一些。修复完成后，最后让它生成一个图标，并打包安装到我电脑上。这就是最终生成的效果。

**04:35** 当我点击鼠标时，会有更多的烟花效果。我们可以看到“新年快乐 2026”的中文字样。每次点到 20 的时候，它会触发一个多重烟花模式。当我鼠标移动到右侧，会看到控制面板，这里有六种烟花的样式，比如星形、双层、树形等。

**05:00** 背景中我们可以隐约看到东方明珠，中间最高的一幢就是上海中心。个人觉得这个生成的效果相当不错，也祝大家 2026 新年快乐。

**05:21** 来看一下，我是如何与它对话的。针对现有的计划文档，我让它进行访谈。我提示 Claude Code 提示它调用 `Spec Interview Skill`（这是我把刚才看到的那段话做成了一个 Skill），然后让它回答我的问题。Claude 随后阅读了计划文档，了解到这是一个关于“冬季小木屋”场景的 Three.js 项目。

**05:41** 项目目标是添加极光天空效果、丰富室内的圣诞装饰，以及实现深度缩放等等。接着，它就开始问我：极光的视觉风格应该是什么样的？室内装饰是什么样的？当相机进入室内时，室外场景应该如何处理？这是我的回复。

**06:05** 再接着，它继续和我探讨了一些新的技术和体验。现在我们看到的这个页面，是我通过 Simo 开源的可视化工具展示的结果。如果我们在 Claude Code 里面实时交互，体验会非常方便，通常就是通过单选或多选来完成访谈。AI 会在一次访谈里提出多个问题让你选择。

**06:24** 这里也就不一一展示了。也就是说，只要给它比较简短的提示，Claude Code 就可以通过多轮提问，让我们的需求更详细，从而做出更好的应用。之后，我将完善后的规格文档发给 o1-pro，让它帮我生成场景。

**06:46** 上方紫色飘动的是极光效果。在这里我们可以看到一个圣诞小屋。滚动鼠标就可以进入圣诞小屋内部，看到圣诞树、壁炉，还有玻璃窗。透过玻璃可以看到极光在闪烁。

**07:08** 之前 Claude Code 问我那个小火车要怎么运行，我提示它按照“8”字形轨迹运行。目前看它生成的轨迹非常标准。此外，它制作的小熊和礼物包装盒细节也非常到位。

**07:30** 我又将同样的规则文档发给 GPT-4o。这是它生成的效果。GPT-4o 在这个问题上的编码效果并没有 o1-pro 好，可以看到有很多闪烁的 Bug。圣诞树不够美观，礼品包装盒也有些问题。

**07:53** 刚才介绍的提示词也可以放在 Codex 里面。比如我在里面做了一个 Skill，提示它阅读 Markdown 文件并问我问题。它接着给出了 15 个问题。虽然 Codex 本身倾向于回答简短，但有了这个“深度访谈”的 Skill Push 它一下，它就能问出非常有深度的问题了。

**08:36** 效果还是非常不错的。以上就是今天介绍的所有内容，推荐大家尝试使用这样的 Prompt 技巧。
